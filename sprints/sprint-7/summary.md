## Risk notes

- **R-sprint-7.1 — same-file contention on reference.md.** 10 of 13 stories edit `skills/ui-audit/reference.md`. Sequential in-session execution (sprint-6 pattern) sidesteps merge hell. If parallelizing to workers is ever revisited, split reference.md into per-phase files first.
- **R-sprint-7.2 — cookie injection for fast-role-switch.** S7-010 documents `document.cookie = ...` restoration but notes HttpOnly session cookies can't be injected that way. Default path (scripted login per transition) is the reliable fallback. Low risk; documented.
- **R-sprint-7.3 — ETA gate on interactive sessions.** S7-013's `--yes`/`--ci` gate depends on CLI flag wiring in SKILL.md arg-parse. If Bash arg-parse doesn't accept `--yes`, the gate degrades to "prompt-and-hang" — which loops via CLAUDE_CODE_AUTONOMY=full would bypass cleanly, but interactive users might hit it. Default autonomy = full in loop mode sidesteps.
- **R-sprint-7.4 — event fixture reliability.** S7-008 scrapes `<script>` blocks via grep. If a future edit to fixture-app.html uses multi-line script literals, the grep pattern breaks. Low risk; the fixture is bundled with the skill and versioned together.
